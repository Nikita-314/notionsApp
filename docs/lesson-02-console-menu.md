# Урок 2. Консольное меню и ввод данных

В этом шаге мы сделали прототип Remka интерактивным.

Теперь приложение умеет:

- показывать список транспорта;
- добавлять новый транспорт через консоль;
- открывать карточку выбранного транспорта;
- выходить из программы.

## Главная идея

До этого данные были просто записаны в коде. Теперь пользователь может вводить их сам.

Это всё ещё не Android-интерфейс, но логика похожа:

- пользователь выбирает действие;
- приложение читает ввод;
- создаёт объект;
- сохраняет его в репозиторий;
- показывает результат.

## Kotlin: while

`while (true)` создаёт бесконечный цикл.

```kotlin
while (true) {
    println("1. Показать транспорт")
    println("0. Выход")
}
```

Так работает главное меню: оно показывается снова и снова, пока пользователь не выберет выход.

## Kotlin: when

`when` похож на более удобный `switch` из других языков.

```kotlin
when (readln().trim()) {
    "1" -> printVehicleList(repository.getVehicles())
    "0" -> return
    else -> println("Неизвестная команда.")
}
```

Мы смотрим, что ввёл пользователь, и выполняем нужный код.

## Kotlin: readln

`readln()` читает строку из консоли.

```kotlin
val name = readln().trim()
```

`trim()` убирает пробелы в начале и конце строки.

## Kotlin: toIntOrNull

Когда пользователь вводит число, на самом деле мы сначала получаем строку.

```kotlin
val number = value.toIntOrNull()
```

Если строка похожа на число, Kotlin вернёт `Int`.
Если нет, вернёт `null`.

Это безопаснее, чем `toInt()`, потому что программа не падает при неправильном вводе.

## Kotlin: return внутри функции

В меню команда выхода выглядит так:

```kotlin
"0" -> {
    println("До встречи!")
    return
}
```

`return` завершает функцию `runMenu`, а значит и программу.

## Kotlin: маленькие функции

Мы вынесли ввод в отдельные функции:

- `readVehicle`;
- `readVehicleType`;
- `readRequiredText`;
- `readOptionalText`;
- `readOptionalInt`;
- `readOptionalLong`.

Так код легче читать. Главная функция меню не знает деталей ввода года или пробега, она просто вызывает `readVehicle`.

## Почему это полезно для Android

В Android вместо `readln()` будут поля ввода на экране.

Но смысл останется тем же:

```text
экран -> пользователь ввёл данные -> создаём Vehicle -> сохраняем в repository
```

Сейчас мы учим именно эту логику без лишней сложности интерфейса.

## Как проверить

Запусти:

```bash
./gradlew run
```

Попробуй:

1. выбрать `1` и посмотреть список;
2. выбрать `2` и добавить транспорт;
3. выбрать `3` и открыть карточку;
4. выбрать `0` и выйти.

## Если русские буквы отображаются неправильно

Это проблема кодировки консоли. Проект настроен на UTF-8 в `build.gradle.kts`:

```kotlin
applicationDefaultJvmArgs = listOf(
    "-Dfile.encoding=UTF-8",
    "-Dsun.stdout.encoding=UTF-8",
    "-Dsun.stderr.encoding=UTF-8"
)
```

Если в IntelliJ IDEA всё равно видны нечитаемые символы, проверь настройки:

- `Settings | Editor | File Encodings`;
- `Global Encoding`: UTF-8;
- `Project Encoding`: UTF-8;
- `Default encoding for properties files`: UTF-8.

После изменения настроек лучше перезапустить запуск `./gradlew run`.
